小结

​ 1) 双方用的json工具处理类不一致，导致解析出错，最终统一为fastJson

​ 2) fastJson版本太老，导致double类型精度缺失，经确认升级为最新版本

​ 3) 对参数需要进行默认排序, json转换的时候追加Feature.OrderedField属性
